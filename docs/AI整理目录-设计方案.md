## 一句话定位

不做「全自动整理」（风险大、信任低），做**「AI 提案、人批准、翻箱执行」**的三段式：AI 只动嘴，动手前每一步都过人。

## 为什么不做全自动

文件移动是高风险操作：移错一个正在被 agent 引用的项目文件，比乱放一百个文件代价更大。市面上自动整理工具（Hazel 等）靠死规则，AI 整理工具靠模型直接 mv——前者不懂语义，后者出错没有回路。翻箱的差异化恰恰是「看得见的信任」，整理也要延续这个哲学。

## 三段式流程

### 1. 扫描与提案（只读）

- 用户在某个文件夹（桌面/下载是主场景）点「AI 整理」
- 翻箱收集**元数据**：文件名、类型、大小、时间、来源 app（macOS 的 kMDItemWhereFroms）——默认不读文件内容，隐私和速度都可控
- 把元数据清单交给本机的 Claude Code（内嵌终端跑 `claude -p` 一次性任务，零新增依赖、用户已有订阅），让它输出**结构化整理提案** JSON：
  ```json
  { "moves": [{ "from": "截屏2026-06-01.png", "to": "_archive/截图/2026-06/", "reason": "三个月前的截图" }],
    "groups": [...], "uncertain": [...] }
  ```
- 提案规则带上花叔的文件管理铁律：**只归档不删除**（一律移入 `_archive/`，>50MB 视频除外，单独列出来问）、项目制（散文件按语义归入项目文件夹或新建项目文件夹）

### 2. 审批界面（核心交互）

- 提案渲染成**勾选清单**：每条 = 文件 → 目标位置 + 一句理由，默认全选，可单条取消
- 「不确定」分组单独标黄，默认**不选**
- 一键「全部跳过某类」（比如：截图不动）

### 3. 执行与回滚

- 翻箱自己执行 mv（不让 AI 碰文件系统），执行前把完整 moves 清单写成**回滚日志** `~/.fanbox/organize-log/<时间戳>.json`
- 整理完成页提供「一键全部撤销」（按日志反向 mv），保留 30 天
- 执行过程在界面可见（逐条打勾），出错单条跳过不中断

## 范围控制（v1 刻意不做）

- 不做定时自动整理（高风险低信任）
- 不做全盘扫描（只对用户指定的单个文件夹）
- 不做文件内容语义分析（v1 只看元数据；内容级语义留给 2.0 的素材索引统一解决）
- 不做重复文件检测（独立功能，另议）

## 待花叔拍板的三个问题

1. **AI 引擎**：用内嵌终端跑 `claude -p`（零依赖、走你的订阅）还是接 API key（更快但要配置）？我倾向前者。
2. **主场景**：先做桌面 + 下载两个预设入口，还是任意文件夹通用入口？我倾向通用入口 + 这两个置顶推荐。
3. **归档目标**：`_archive/` 直接建在被整理的文件夹内（你的现行惯例），还是集中到一个全局归档区？我倾向就地 `_archive/`，和你手动整理的习惯无缝。
