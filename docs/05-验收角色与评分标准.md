# 翻箱 FanBox · 验收角色与评分标准

版本 v1.0 · 2026-06-09

## 用法

每完成一个开发阶段，由 **5 个独立 subagent** 分别扮演下列角色，审阅「成品 + 真机实跑截图」后各自打分（写产品的 AI 不给自己打分）。每个角色：

1. 按自己的 5 个维度打分（各 20 分，满分 100）；
2. 明确列出「**阻碍达到 90 分的具体项**」；
3. 触发任一 **红线（硬否决）** 直接判不通过，无论总分。

**通过标准**：五个角色全部 ≥ 90 分，且无单项 < 14 分，且无红线触发。未达标 → 据清单迭代 → 重新评，直到全部达标。

维度覆盖：可用性（③主 + ①）· 视觉审美（②主）· 功能体验（①④主）· 健壮性底线（⑤贯穿）。

---

## ① 重度 vibe coder（花叔本人画像）—— 自用准星

> **你是谁**：你是花叔，AI Native Coder，一下午起十个项目，同时跑着 Claude Code / Codex。你对「摩擦」零容忍，对「省一步」极度敏感。这工具存在的唯一理由，是让你比「Finder + iTerm + 浏览器」三件套更快——省不掉一次切换，就是失败。

| 维度 | 看什么 | 20 分 |
|---|---|---|
| 找回 | 忘了名字的项目/文件，多快能找回（⌘K 文件夹搜索准不准、快不快） | |
| 验货 | agent 改了啥，能不能不切窗口就看到（预览/实时刷新） | |
| 抵达 | 从「看到」到「在终端/编辑器动手」几步 | |
| 联动 | 文件 × 终端 × 预览是不是一个有机整体，而非三个面板硬拼 | |
| 心流 | 整体用下来，你愿不愿意把 iTerm 关掉 | |

**90 门槛**：核心循环（找项目 → 开终端跑 agent → 看它改文件）全程不离开翻箱，且明显快于原工作流。
**红线**：任何一步让你心里冒出「还不如直接开 iTerm」。

## ② macOS 原生审美设计师 —— 视觉审美守门人

> **你是谁**：你打磨过 Linear / Things / Warp 级别的产品，对「web 套壳味」生理性厌恶。你相信留白是信息不是浪费，动效要有物理感，每个像素都有理由，克制是最高级的设计。

| 维度 | 看什么 | 20 分 |
|---|---|---|
| 原生质感 | vibrancy / 标题栏 / 圆角 / 阴影像不像真 mac app | |
| 排版节奏 | 字体层级、间距、对齐的呼吸感 | |
| 配色品味 | 三皮肤是否高级，终端区与皮肤是否和谐 | |
| 动效 | 过渡 / 拖拽 / 面板切换是否顺滑、有物理感 | |
| 一致性 | 图标 / 控件 / 状态语言是否统一 | |

**90 门槛**：截图发到 Dribbble / Twitter 不露怯，看不出是「网页套的壳」。
**红线**：生硬阴影、廉价渐变、动效突兀或缺失、终端区与整体割裂。

---

## ③ 首次上手的新用户 —— 可用性（可学习性）

> **你是谁**：你第一次打开翻箱，没看任何文档，但你懂技术。你相信好工具不需要说明书，核心功能应该在第一分钟自己浮现。

| 维度 | 看什么 | 20 分 |
|---|---|---|
| 可发现性 | 核心入口（搜索 / 终端 / 编辑 / 新建）一眼能找到 | |
| 即时反馈 | 每个操作有清晰反馈，不让你猜成没成 | |
| 无错可恢复 | 误操作有护栏，删除能撤 | |
| 直觉一致 | 交互符合直觉（单击预览 / 双击打开 / 右键菜单） | |
| 第一分钟成功 | 不看文档完成「找文件 → 开终端」 | |

**90 门槛**：零文档下，第一分钟独立完成核心任务，全程无困惑卡点。
**红线**：需要猜的交互、没反馈的操作、藏太深的核心功能。

---

## ④ 终端重度用户 / 资深工程师 —— 功能体验（终端技术深度）

> **你是谁**：你在终端里活了十年，vim / tmux / claude code 是肌肉记忆。你一眼能看出「真终端」还是「玩具终端」。终端不顺手，一切免谈。

| 维度 | 看什么 | 20 分 |
|---|---|---|
| TUI 保真 | claude code / vim / htop 等 TUI 渲染正确、不花屏 | |
| 交互完整 | ctrl-c / ctrl-d、复制粘贴、滚动、resize 重排 | |
| 中文与颜色 | 中文不乱码，256 色 / 真彩色正确 | |
| 性能 | 大量输出不卡，无明显延迟 | |
| 多会话 | 多标签互不干扰、cwd 正确、关闭不泄漏进程 | |

**90 门槛**：愿意拿它当主力终端，跑 claude code 一整天不想切回 iTerm。
**红线**：花屏、乱码、resize 崩、粘贴丢字符、明显卡顿、关标签后进程泄漏。

---

## ⑤ 破坏性可靠性质量官 —— 健壮性与数据安全底线

> **你是谁**：你专门找崩溃和数据丢失，假设用户会做最蠢的操作。你相信再好看再好用，丢一次数据就归零——护栏比功能重要。

| 维度 | 看什么 | 20 分 |
|---|---|---|
| 数据安全 | 删除可恢复（废纸篓）、编辑不丢内容、写入不损坏 | |
| 边界处理 | 大文件 / 超长目录 / 特殊字符 / 符号链接不崩 | |
| 失败优雅 | 断网 / CDN 挂 / node-pty 未编译 / 端口占用都有降级提示 | |
| 资源管理 | 多终端、长时间运行不内存泄漏，关窗清理 pty | |
| 误操作护栏 | 危险操作可逆或有确认 | |

**90 门槛**：连续 20 个边界 / 破坏性操作，无数据丢失、无崩溃、无静默失败。
**红线**：永久删除无法恢复、编辑覆盖丢内容、崩溃、资源泄漏、静默吞错误。

---

## 阶段适配

标准是**终态产品**的标准。每个阶段验收时，只对「该阶段已交付的能力」打分，未实现的能力标注「待后续阶段」不计扣分——但红线（尤其数据安全）任何阶段都不豁免。

| 阶段 | 验收重点角色 |
|---|---|
| 1 · 内嵌终端 MVP | ④终端深度 · ②审美 · ⑤可靠性 |
| 2 · 三方联动 | ①自用准星 · ③可学习性 |
| 3 · 多终端 + 布局 | ②审美 · ④终端 · ①自用 |
| 4 · Monaco 编辑 | ④（编辑手感）· ③ · ⑤ |
| 终态 | 五角色全员 ≥ 90 |

## 验收结果（桌面版·终端驾驶舱，2026-06-09）

5 个独立 subagent 各扮一角，审「成品 + 真机实跑截图 + 代码」打分，逐轮据「阻碍 90 的具体清单」迭代修复。

| 角色 | R1 | R2 | R3 | R4/R5 | 终评 |
|---|---|---|---|---|---|
| ① 重度 vibe coder | 77 | 79 | 88 | 91 | ✅ |
| ② 原生审美设计师 | 72 | 84 | 88 | 90 | ✅ |
| ③ 零文档新用户 | 88 | 94 | — | 94 | ✅ |
| ④ 终端十年老兵 | 76 | 87 | 92 | 92 | ✅ |
| ⑤ 破坏性质量官 | 74 | 87 | 91 | 91 | ✅ |

**结论：全部 ≥90，无红线触发，开发达标。**

各轮关键修复：
- R1：审美两条红线（换肤硬切撕裂、终端区与皮肤割裂）解除；终端 resize 重排+WebGL+Edit菜单+真重开；数据安全（原子写+并发覆盖保护+trash转义+UTF-8边界）；fs.watch 递归。
- R2：换肤过渡纳入 color；双层阴影；respawn 复用会话目录+reset；变更高亮（agent 改文件即亮）；server 隐患（rename 失败清理 tmp、外部删除判 conflict、trash 授权错误转中文、Linux 降级非递归）。真机验证 vim TUI / 粘贴 50 行 / seq 5000 / respawn 全通过。
- R3：变更高亮带计数（改·N）+tooltip 定位深层改动+单一清理定时器；磨砂玻璃浮层；排版呼吸。
- R4：网格图标类型色磨砂圆角底座，主区静止态去 web-view 感。
