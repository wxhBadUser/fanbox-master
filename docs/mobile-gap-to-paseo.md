# Mobile-R0 FanBox Gap To Paseo

> Scope: 本差距地图以 Paseo 为参考对象，不复制 Paseo 实现。Gap level 代表 FanBox 相对目标体验的缺口，不代表代码质量问题。

## Reading Conclusion

FanBox 已经具备 LAN mobile gateway、pair token、REST/SSE、sessions、files、skills、usage、approvals、devices/audit read model 的雏形。与 Paseo 的根本差距是：Paseo 的 daemon 统一拥有 agent lifecycle、workspace identity、timeline 和 permission，而 FanBox 目前仍由多个本地 JSON store、runner adapter、static UI state 和分散 REST endpoint 拼出 mobile 体验。

因此后续路线应该 backend-first：先把 FanBox 的 mobile contract 做成稳定、可消费、可验证的 source of truth，再谈 mobile UI 重做和远程 relay。

## Priority Interpretation

The table implies this order:

1. Lock the backend contract that already exists.
2. Make desktop-running agents visible and resumable through a stable read model.
3. Make phone-started sessions explicit.
4. Harden device, audit, approval, and pairing safety.
5. Redesign mobile UI around the new contract.
6. Only then study local WebSocket and remote relay.
