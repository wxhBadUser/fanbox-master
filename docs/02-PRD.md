# 翻箱 FanBox — 产品需求文档 (PRD)

版本 v1.0 · 2026-06-09

## 1. 产品定位

vibe coding 的本地文件指挥中心。在浏览器里查看、搜索、预览、一键打开本机的文件夹与文件，专注「快速找回 + 原地验货 + 一键抵达」。

## 2. 技术架构（与「为什么不是纯浏览器插件」）

浏览器扩展受沙箱限制，无法自由读取本机任意目录，更无法在操作系统层面「打开」文件（启动 VS Code、在 Finder 中显示）。因此采用：

```
本地 Node 服务 (仅监听 127.0.0.1)  ←─ HTTP ─→  浏览器单页应用 (UI)
        │
        └─ 完整文件系统读取权限 / 调用系统命令打开文件
```

- **零依赖**：纯 Node 内置模块（http / fs / child_process），不装任何 npm 包。
- **零配置**：`node server.js` 一条命令，自动打开浏览器。
- **本地优先**：只绑定回环地址，数据不出本机；收藏/最近记录存 `~/.fanbox/config.json`。
- **跨平台**：macOS / Windows / Linux 的「打开」命令分别适配（open / start / xdg-open）。

## 3. 功能需求

### 3.1 浏览导航 (P0)
- 文件夹网格视图 / 列表视图切换，状态记忆。
- 面包屑路径导航，逐级可点。
- 后退、上一级按钮；⌘[ 后退快捷键。
- 排序：名称 / 修改时间 / 大小，文件夹恒在前。
- 显示/隐藏「点文件」开关。
- 当前文件夹内即时筛选（`/` 聚焦）。
- 侧边栏快速入口：主目录、桌面、文档、下载、常见代码目录（仅显示存在的）。

### 3.2 全局搜索 (P0)
- ⌘K 唤起命令面板，在「当前目录树」范围内搜索。
- 文件名**模糊匹配**（子序列 + 连续/词首/靠前加权打分），命中字符高亮。
- `内容:` 前缀切换为**全文内容搜索**（grep），返回命中行号与代码片段预览。
- 键盘操作：↑↓ 选择、↵ 进入目录或预览、⌘↵ 直接用编辑器打开。
- 搜索带忽略表（node_modules/.git/dist 等）+ 结果上限 + 时间预算，保证大目录下不卡。

### 3.3 文件预览 (P0)
原地验货，右侧面板内嵌：
- **文本/代码**：语法高亮（highlight.js），显示大小/修改时间/类型。
- **Markdown**：渲染为富文本（marked），代码块高亮。
- **HTML**：默认 iframe 实时渲染成品，可一键切换查看源码。
- **图片**：内嵌大图。
- **视频/音频**：内嵌播放器（支持 Range 流式）。
- **PDF**：iframe 内嵌阅读。
- 富预览依赖的 marked/highlight.js 走 CDN，**离线时优雅降级**为纯文本，不报错。

### 3.4 一键打开 (P0)
预览面板操作区：
- 「默认应用打开」（系统关联程序）。
- 「在编辑器打开」（VS Code `code` 命令，未安装则回退默认）。
- 「在 Finder/资源管理器显示」。
- 「复制路径」。
- 列表/网格内双击文件 = 默认应用打开，双击文件夹 = 进入。

### 3.5 收藏与最近 (P1)
- 任意文件/文件夹可收藏，侧边栏常驻，持久化到本地。
- 「最近打开」：记录通过翻箱打开过的文件，侧边栏快速重开。
- 「最近修改」：扫描当前目录树，按 mtime 倒序列出最新文件（还原「我刚做了啥」）。

### 3.6 项目识别 (P1)
- 进入目录时自动识别项目类型（package.json→NODE，index.html→WEB，pyproject→PYTHON，Cargo.toml→RUST，go.mod→GO，.git→GIT），面包屑旁显示徽章。

## 4. 非功能需求
- **性能**：单目录列出 < 100ms；搜索/grep 有 1.5~3.5s 时间预算与结果上限，绝不无限阻塞。
- **安全**：仅 127.0.0.1 监听；拒绝含空字节的非法路径；静态资源目录穿越防护。
- **健壮**：单个文件读取失败不影响整体列表；CDN 失败不影响核心功能。

## 5. 交互与快捷键
| 操作 | 快捷键 |
|---|---|
| 全局搜索 | ⌘K / Ctrl+K |
| 当前目录筛选 | `/` |
| 搜索结果导航 | ↑ ↓ |
| 打开 / 预览 | ↵ |
| 在编辑器打开 | ⌘↵ |
| 后退 | ⌘[ |
| 关闭面板 | Esc |

## 6. 不做什么（范围边界）
- 不做文件增删改、重命名、批量移动（交给 Finder，避免误操作风险）。
- 不做云同步、不做多用户、不做账号体系。
- 不做远程访问（坚持本地、回环地址）。

## 7. 验收标准
本 PRD 的最终验收 = 第 01 号文档定义的五个用户角色，对「概念 + PRD + 成品」综合评分均 ≥ 90 分。
