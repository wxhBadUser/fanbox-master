# 翻箱 FanBox · 桌面版路线图（Electron · 终端驾驶舱）

版本 v0.2 · 2026-06-09 · 规划稿
（v0.1 重心在 Monaco 编辑器；v0.2 重心调整为「内嵌终端 + coding agent 联动」，编辑退为锦上添花）

## 北极星

> 翻箱不是文件管理器，也不是 IDE。它是 **vibe coding 的驾驶舱**。

花叔一下午起十个项目，每个里跑着一个 Claude Code / Codex / Aider。痛点：agent 散在十个终端窗口、文件散在各处、改了啥要来回切。翻箱把「**看文件 / 验成果**」和「**终端里的 agent**」收进同一界面并联动。

**自用优先**是最高准则——一切取舍以花叔真实工作流为准星，满足不了自己就毫无意义。

### 做 / 不做（守住边界）

- ✅ 做：文件浏览/搜索/预览、内嵌终端跑 TUI agent、终端×文件×预览三方联动、轻量原地编辑、私人皮肤
- ❌ 不做：插件市场、调试器、LSP 自动补全、完整 IDE 工程化——那些 agent 自己干，或一键甩 Cursor

## 架构

```
Electron
├─ 主进程 main
│   ├─ 复用 server.js 的文件能力（list/search/read/write/trash/rename/create/open）
│   └─ node-pty：为每个终端开真 PTY，承载 Claude Code 等 TUI agent
└─ 渲染进程 renderer：现有 public/ UI ＋ xterm.js 终端 ＋（后续）Monaco
        三栏：左 文件树/导航 · 中 预览/编辑 · 右(或下) 终端
        联动：三方共享 cwd；agent 改文件 → fs.watch → 预览自动刷新
```

接后端方式：先 `require('./server.js')` 跑 localhost（零改造、1 天可运行），再渐进迁到 IPC（`ipcMain.handle`）去掉 HTTP、关网络面。终端走 IPC + node-pty（不经 HTTP）。

## 分阶段（每阶段独立可交付、可停）

### 阶段 0 · 现状（已完成）
浏览器版：浏览 / 搜索（含文件夹）/ 预览 / 轻量编辑 / 新建 / 重命名 / 移废纸篓 / 开终端 / 后台常驻。

### 阶段 1 · Electron 壳 + 内嵌终端 MVP（约 2–3 天）★核心
- Electron 套壳，加载现有 UI（方式 a）。
- 右侧（或底部）面板嵌 `xterm.js`，主进程 `node-pty` 起真 shell。
- 能在界面里开终端、跑起 Claude Code / Codex 这类 TUI agent，输入输出顺滑、支持 resize、颜色正确。
- 终端默认 cwd = 当前浏览目录（第一层联动）。
- 交付：**在翻箱里开终端、跑起 agent、左边看文件**——核心爽点立现。

### 阶段 2 · 三方联动（约 2–3 天）★核心
- 点文件夹 → 终端 `cd` 过去（或新开一个该目录的终端）。
- `fs.watch` 当前目录树，agent 改了文件 → 预览面板/列表自动刷新（看着 agent 干活）。
- 终端里 `cd` → 左侧文件树跟着定位（双向，按需）。

### 阶段 3 · 多终端 + 布局（约 3–4 天）
- 终端标签栏：每个项目一个 agent session，常驻不丢。
- 三栏布局可拖拽调比例、可折叠终端区；布局记忆。
- 原生菜单栏 / Dock / 窗口尺寸记忆。

### 阶段 4 · Monaco 轻量编辑升级（约 1–2 天）
- 预览面板 textarea → Monaco：语法高亮、⌘F 查找替换、多光标。
- 定位仍是「轻改 + 验货」，不做完整 IDE。

### 阶段 5 · 按需深化（开放扩展位）
- 会话记忆（重开 app 恢复终端/项目）、Git diff 只读视图、agent 快捷指令面板、「在 Cursor 打开到第 N 行」。
- 面板式架构预留：未来可挂新面板（日志/任务/预览变体）而不动核心。

## 审美方向（mac 原生质感）

参考坐标：**Warp**（现代终端审美标杆）、**Linear**（克制精致）、**Things**（原生质感）、**Zed**。

- 原生：vibrancy 毛玻璃侧栏、SF 字体、红绿灯区留白、合理圆角与阴影，去掉「网页套壳感」。
- 延续现有三皮肤——尤其「终端 · Volt 荧光绿」和内嵌终端区是天作之合；终端配色与皮肤联动。
- 终端区：等宽字（JetBrains Mono / SF Mono）、舒适行距、光标与选区配色随皮肤。

## 关键取舍与风险

- **node-pty 是原生模块**：需对应 Electron 版本编译（`electron-rebuild`），首次配置会踩半天坑，之后稳定。
- **包体**：Electron ~120MB（自用无所谓）。
- **告别「零依赖」**：上 Electron + xterm + node-pty + （后续）Monaco，零依赖哲学终结——定位升级的必然代价。
- **公证**：分发他人需 Apple Developer（$99/年）签名+notarize；自用右键打开即可。
- **安全**：终端有完整 shell 权限（自用工具，本就是要的能力）；仍只本机，无远程。

## 工作量总览（单人粗估）

| 阶段 | 内容 | 估时 |
|---|---|---|
| 1 | Electron 壳 + 内嵌终端 MVP | 2–3 天（首配 node-pty 多半天） |
| 2 | 三方联动（cwd 共享 + 文件监听刷新） | 2–3 天 |
| 3 | 多终端 + 布局 | 3–4 天 |
| 4 | Monaco 轻量编辑 | 1–2 天 |
| 5 | 按需深化 | 持续 |

核心（阶段 1+2）约 **1 周**，就有「能在界面里指挥 coding agent 的翻箱驾驶舱」。

## 推进建议

阶段 1（先摸到「界面内跑 agent」的爽感）→ 阶段 2（联动）→ 用一阵 → 3/4/5 按需。每阶段独立可用、随时可停，不梭哈。
