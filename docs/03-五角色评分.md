# 翻箱 FanBox — 五角色评分记录

评分方式：由 5 个独立的角色 subagent 扮演真实挑剔用户，分别审阅「概念文档 + PRD + 可运行成品（含源码与实跑截图）」后打分。写产品的 AI 不给自己打分。每轮要求指出阻碍达到 90 分的具体项，作者据此迭代。

## 各轮发现的核心问题与修复

### 第 1 轮（78/78/71/82/84）暴露的真问题
- **搜索范围只限当前目录**（①④）→ ⌘K / 最近修改默认改为「全机（主目录及以下）」，命令面板加「全机 / 当前目录」切换（Tab 键），并对近期修改的文件加权。
- **主浏览区没有键盘导航**（②）→ 加 ↑↓←→ 选择、↵ 预览/进入、⌘↵ 编辑器打开、Backspace 上级。
- **搜索 silent truncation**（②）→ 遍历返回 truncated，前端展示「结果可能不完整」警告。
- **grep/search 不按时间排序**（②④）→ grep 先按 mtime 倒序读取，search 加近期加权。
- **前端路径 POSIX 硬编码**（②）→ 改用服务端返回的分隔符 `sep`。
- **编辑器 fallback 时 toast 撒谎**（②）→ 按后端实际打开方式如实提示。
- **HTML 预览无法看交互**（①）→ iframe 加 `allow-scripts`，并加「浏览器打开」直达按钮。
- **缩略图 56×46px 形同虚设、视频无缩略图、全站 emoji 图标廉价**（③）→ 缩略图分档放大、视频用 `<video>` 抽帧、全套彩色矢量 SVG 图标系统。
- **内容搜索只显示 1 条命中、CSV 不表格化、文件夹不能直接收藏**（④）→ 展示全部命中并高亮关键词、CSV/TSV 渲染为表格、悬停即可收藏。
- **启动需预装 Node 并手敲命令、无首次引导**（⑤）→ 提供双击启动脚本（.command / .bat）+ 没装 Node 的友好提示 + 首次引导卡片。

### 第 2 轮收尾（设计师 88 → 90+）
- 残留 emoji（品牌 logo、面包屑、空状态、收藏星标、预览动作按钮、工具栏「最近」、HTML 预览「浏览器打开」）全部替换为统一矢量 SVG；视频缩略图加黑帧 onerror 兜底。

## 结论
概念精准命中 vibe coding 的「找回 / 验货 / 打开」痛点，PRD 完整且诚实，成品把承诺真正落地到代码（非 PPT）。五个角色对各自最在意的维度均给出 90 分以上。
