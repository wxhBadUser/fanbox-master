# 翻箱 FanBox — 概念与用户研究

## 一、为什么 vibe coding 时代需要一个新的文件查看器

vibe coding 改变了文件的「生产方式」，也就改变了文件管理的需求。

过去你写代码，文件是你一个一个手敲出来的，路径在脑子里很清楚。现在你跟 AI 对话，一句话生成一个完整项目，一个下午能起五六个 demo。文件量暴涨，但你对它们的「位置记忆」几乎为零——你记得「我昨天让 AI 做了个落地页」，但不记得它叫什么、在哪个文件夹。

系统自带的 Finder / 资源管理器是为「我知道文件在哪」设计的，它假设你有清晰的目录心智模型。vibe coder 没有。我们要的不是更强的「浏览」，是更强的「找回」和「快速验货」。

三个真实痛点：

1. **散**。Claude Code、Cursor、各种 AI 工具把项目甩在不同地方：`~/Projects`、`~/Desktop`、`~/Documents`、临时目录。没有统一入口。
2. **认不出**。AI 生成的文件名经常是 `index.html`、`app.py`、`untitled`，光看名字分不清是哪个项目的产物。需要「打开看一眼」才知道是不是要找的东西，但 Finder 的预览又弱。
3. **打开慢**。找到了还要右键决定用什么打开：浏览器看 HTML？VS Code 改代码？预览看图？每一步都打断心流。

所以翻箱的定位是：**vibe coding 的本地文件指挥中心**。不是替代 Finder，是补上 Finder 缺的那块——快速找回、原地验货、一键打开。

## 二、设计原则

- **找回优先于浏览**：⌘K 全局模糊搜索是第一公民，不是藏在角落的功能。
- **原地验货**：预览要足够强，看一眼就知道是不是要找的，不用真打开应用。Markdown 渲染、HTML 实时预览、代码高亮、图片/视频/PDF 内嵌。
- **一键抵达目的地**：找到后，无论是浏览器、编辑器还是 Finder，一下点到位。
- **本地、零配置、零依赖**：一条命令起服务，不上传任何数据，不装任何包。
- **键盘优先**：⌘K 搜索、↑↓ 选择、↵ 打开、⌘↵ 在编辑器打开、`/` 当前目录筛选。

## 三、五个用户角色

每个角色都对应 vibe coding 时代一类真实的人。我们的目标是让这五个人用完都打 90 分以上。

### 角色 1：花叔型「非技术创作者 / AI Native Coder」
- **画像**：不写代码，全靠 AI 生成产品。一周起十几个 HTML demo、小工具、落地页。
- **核心诉求**：「我上周做的那个东西去哪了？」需要按记忆里的关键词找回，需要 HTML 能直接在浏览器里预览成品，而不是看源码。
- **打分关键**：⌘K 找得到模糊记忆的项目；HTML 一键渲染预览；最近修改列表能还原「我刚做了啥」。

### 角色 2：独立开发者 / Indie Hacker
- **画像**：同时维护 5~10 个 repo，技术熟练，效率敏感。
- **核心诉求**：闪电般在项目间跳转，秒开 VS Code，按文件名/内容快速定位。讨厌任何卡顿和多余点击。
- **打分关键**：搜索速度；⌘↵ 直接编辑器打开；项目类型识别（看到 NODE/PYTHON 徽章）；node_modules 不拖慢搜索。

### 角色 3：设计师 / 内容创作者
- **画像**：管理大量图片、设计稿、导出的封面、视频素材。
- **核心诉求**：视觉化浏览，缩略图网格，快速翻图找素材，预览大图和视频。
- **打分关键**：网格视图的图片缩略图；图片/视频/PDF 内嵌预览；按时间排序找最新导出的图。

### 角色 4：AI 产品经理 / 研究者
- **画像**：囤大量 Markdown 笔记、调研文档、prompt、数据集。
- **核心诉求**：渲染阅读 Markdown，搜文件「内容」而不只是文件名（记得写过某句话，忘了在哪个文件）。
- **打分关键**：Markdown 富文本渲染；「内容:」全文搜索；收藏常用文档。

### 角色 5：学习者 / 学生
- **画像**：跟教程做练习，整理课程资料、代码作业，目录结构混乱。
- **核心诉求**：简单直观不学习成本，能收藏常去的文件夹，看代码有高亮。
- **打分关键**：零上手成本；侧边栏快速入口与收藏；代码语法高亮；面包屑导航清晰不迷路。

## 四、竞品与差异

| | Finder / 资源管理器 | VS Code 文件树 | 翻箱 FanBox |
|---|---|---|---|
| 全局模糊搜索 | 弱（Spotlight 混杂） | 限当前工作区 | ⌘K 任意根目录 |
| 内容搜索 | 慢/需索引 | 限工作区 | 「内容:」即时 grep |
| 富预览 | 弱 | 需打开文件 | MD渲染/HTML预览/代码高亮内嵌 |
| 一键多方式打开 | 右键多步 | 仅编辑器内 | 浏览器/编辑器/Finder 一键 |
| 项目识别 | 无 | 无 | 自动徽章 |
| 为 vibe coding 优化 | 否 | 否 | 是 |

翻箱不跟 Finder 拼「文件操作」（剪切、重命名、批量移动），它专注「查看 + 找回 + 打开」这条链路，做到极致顺手。
